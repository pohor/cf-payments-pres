Consumer Financing
Provider-Agnostic Frontend Components

A shared UI layer that lets HCP surface financing options to customers across every touchpoint - with any lending partner, without rebuilding the interface.

Every time HCP adds a new lending partner, the CF team rebuilds the same UI across 3 codebases (hcp-web, home-base, payments-components) and 20 placements. With MoneyLion, that cost ~100 story points. With each new partner (Synchrony, Wells Fargo, GoodLeap), that cost repeats. Provider-agnostic components eliminate that.

This is an infrastructure improvement — Pros and homeowners see no visible change. The benefit is that new financing options and updated terms reach all 20 surfaces faster.
Why now

Synchrony passive distribution (~42 SP across 8 surfaces) is the next major CF epic and assumes agnostic components already exist. Without this work, Synchrony is blocked — and every future provider (Wells Fargo, GoodLeap) requires a full ~100 SP rebuild.

How it works

Consumer Financing Provider-Agnostic Frontend Components

These are frontend UI components that display Consumer Financing call-to-actions, disclaimers, and informational content across all customer and pro-facing surfaces.

One component. Any provider.

Each component is a provider-agnostic shell with named slots (logo, promo text, status chip, CTA). The backend returns a provider config object with the content for each slot - branding, copy, rates, and actions. The component renders whatever config it receives, so swapping Wisetack for Synchrony or adding a new lender means updating the config, with almost no frontend code changes.
